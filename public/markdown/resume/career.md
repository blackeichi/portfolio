## Key Projects' TOP3

### 1. 회사 메인 상품(근태관리+출입문 보안 시스템)의 리뉴얼

**기간**: 2024.01 ~ 2024.03

#### 📋 프로젝트 요약
기존 레거시 코드베이스는 기능이 지속적으로 추가되며 구조적 복잡도가 증가했고, 사용자 인터랙션 시 UI 버벅거림(lag) 등 성능 문제도 발생하고 있었습니다. 
또한 UI 구성의 일관성이 부족하여 내부 및 고객사로부터 사용자 경험 개선에 대한 요구가 지속적으로 제기되었습니다. 이에 단순 유지보수가 아닌 프론트엔드 구조 개선과 UI/UX 리뉴얼이 필요하다고 판단하여 팀을 설득하고 프로젝트를 주도적으로 진행하였습니다.

#### ✅ Action
**아키텍처 개선**

- 클래스형 컴포넌트를 함수형 컴포넌트로 전환하여 코드 가독성과 유지보수성 개선
- 커스텀 훅을 설계하여 로딩 / 요청 / 에러 / 재시도 로직을 공통화
- 공통 리소스와 라우트별 리소스를 분리하여 프로젝트 구조 재설계 및 모듈화

**성능 최적화**

- 메모이제이션 및 컴포넌트 책임 분리를 통해 불필요한 리렌더링 최소화
  - 사용자 인터랙션 시 발생하던 UI 렉 문제 해결
- 코드 분할(dynamic import) 및 이미지 최적화를 적용하여 초기 로딩 성능 개선
- Lighthouse 주요 지표 CLS, LCP 개선 (성능 지표 65 → 92 개선)

**코드 품질 개선**

- 동일한 역할의 라이브러리를 하나로 통일하여 의존성 단순화
- 불필요한 코드 제거 및 분산된 로직 정리 리팩토링
- 오래된 라이브러리를 최신 버전으로 교체하여 보안성과 유지보수성 향상

**UI/UX 개선**

- 최신 트렌드에 맞는 UI 리뉴얼
- 폰트 및 색상 체계 통일로 가독성 향상
- 레이아웃 재정리로 불필요한 여백 제거 및 직관성 개선
- 사용자 경험 개선
  - 데이터 로딩 상태 표시
  - 반복 작업 최소화
  - 자연스러운 애니메이션 적용

### 2. 코나아이 근태 관리 웹 애플리케이션

**기간**: 2025.07 ~ 2025.12

#### 📋 프로젝트 요약
근태 관리 시스템 구축 프로젝트로, 관리자용 웹과 직원용 웹 애플리케이션 두 개의 서비스를 동시에 개발했습니다.
직원은 휴가 및 근무 일정 요청, 개인 근태 기록 조회 등을 할 수 있고, 관리자는 데이터 관리 및 승인 업무를 수행할 수 있도록 설계되었습니다.
약 300명 이상의 직원이 사용하는 서비스로 다양한 사용자 환경을 고려한 UX와 안정적인 동작이 중요했으며, 특히 대량의 근태 데이터를 테이블로 조회하는 기능에서 렌더링 성능 최적화를 진행했습니다.

#### ❗ Problem
- 테이블에 수천 건의 데이터가 동시에 렌더링되며 성능 저하 및 OOM 가능성 존재
- 대용량 데이터 처리 시 불필요한 리렌더링과 상태 업데이트로 인한 성능 저하 우려
- Virtualization, Intersection Observer 도입을 검토했으나
  - Virtualization 적용 시 브라우저 기본 검색(Ctrl + F) 사용 불가
  - Intersection Observer는 테이블 구조에 적용하기 어려운 제약 존재
- 다수의 직원이 사용하는 서비스 특성상 다양한 사용 상황과 예외 케이스 대응 필요

#### ✅ Action
**아키텍처 및 개발 환경 구축**
- Next.js + TypeScript 기반 프로젝트 환경 구축
- 향후 SSR 확장을 고려한 구조 설계

**테이블에서 대용량 데이터 처리 최적화**
- 데이터 처리 로직의 연산 비용을 줄이기 위한 로직 개선
- useState / useEffect 사용 최소화로 불필요한 상태 업데이트 감소
- 컴포넌트 책임 분리 및 메모이제이션을 통해 불필요한 리렌더링 방지
- 스타일 및 컴포넌트 구조 경량화로 렌더링 비용 최소화

**사용자 경험 및 안정성 개선**
- 다수의 직원이 사용하는 서비스 특성을 고려해 예외 처리 및 사용자 피드백 로직 강화
- 오류 상황에서도 사용 흐름이 끊기지 않도록 로딩 상태, 오류 메시지, 인터랙션 처리 개선

### 3. 현대중공업 식수관리 시스템

**기간**: 2023.02 ~ 2023.08

#### 📋 프로젝트 요약
현대중공업 사업장의 식당 운영 현황을 관리하는 실시간 식수 관리 시스템 개발 프로젝트입니다.
MQTT 기반으로 식당 단말기에서 전달되는 데이터를 수신하여 수천~수만 명 규모의 식사 데이터를 실시간으로 모니터링할 수 있는 대시보드를 구현했습니다.
또한 식당 정보 관리, 단말기 앱 업데이트 관리 등 운영자가 설비 상태와 식당 운영 현황을 실시간으로 확인하고 관리할 수 있는 관리자 시스템입니다.

#### ❗ Problem
- 여러 사업장의 식당에서 발생하는 대규모 식사 데이터를 실시간 모니터링 필요
- 식당 단말기 상태 및 데이터가 지속적으로 업데이트되며 실시간 UI 반영 필요
- 식당 운영 정보(메뉴, 정책, 상태 등)를 실시간으로 업데이트 및 관리할 수 있는 시스템 필요
- 단말기 앱 업데이트 진행 상황을 실시간으로 확인할 수 있는 관리 기능 필요

#### ✅ Action
**실시간 데이터 모니터링**
- MQTT 기반 실시간 데이터 수신 로직 구현
- 식당 단말기에서 전달되는 데이터를 파싱하여 실시간 식수 카운트 및 상태 정보를 대시보드에 반영

**운영 대시보드 구현**
- 식당별 실시간 식수 카운트 및 온라인/오프라인 상태 표시 기능 개발
- 운영자가 식당 상태를 직관적으로 확인할 수 있도록 대시보드 UI 구현

**관리자 기능 개발**
- REST API 기반 식당 정보 관리 기능 (추가 / 수정 / 삭제) 구현
- 식당 메뉴 및 정책 정보 관리 기능 개발

**단말기 관리 기능 구현**
- 식당 단말기 앱 업데이트 기능 및 실시간 Progress Bar UI 구현
- 단말기 업데이트 상태를 실시간으로 확인할 수 있는 관리 화면 개발
