

# 🚀 Key Projects

## 1. 근태관리 + 출입 보안 시스템 웹 리뉴얼

### 📅 기간

2024.01 ~ 2024.03

### 📌 프로젝트 개요

회사 메인 웹 애플리케이션의 레거시 구조 개선과 UI/UX 리뉴얼 프로젝트를 주도적으로 진행했습니다.  
기존 코드베이스는 기능 확장 과정에서 구조적 복잡도가 증가했으며 UI 렌더링 성능 문제와 사용자 경험 이슈가 지속적으로 발생하고 있었습니다.

### ⚠️ 주요 문제

- 클래스 기반 레거시 코드 구조로 인한 유지보수 어려움
- 사용자 인터랙션 시 UI 렉 발생
- UI 디자인 및 일관성 부족

### 💡 해결

#### 아키텍처 개선

- 클래스형 컴포넌트를 함수형 컴포넌트 구조로 전환
- 로딩 / 요청 / 에러 / 재시도 로직을 Custom Hook으로 공통화
- 공통 리소스와 라우트 리소스를 분리하여 프로젝트 구조 재설계

#### 성능 최적화

- 메모이제이션 및 컴포넌트 책임 분리를 통해 불필요한 리렌더링 제거
- dynamic import 기반 코드 스플리팅 적용
- 이미지 최적화 및 렌더링 구조 개선

#### UI/UX 개선

- UI 디자인 시스템 정리 및 스타일 일관성 확보
- 로딩 상태 표시 및 인터랙션 개선

### ✨ 성과

- Lighthouse 성능 점수 **65 → 92** 개선
- 사용자 인터랙션 시 발생하던 UI 렉 문제 해결
- 코드 구조 개선을 통한 유지보수성 향상

## 2. 코나아이 근태 관리 웹 애플리케이션

### 📅 기간

2025.07 ~ 2025.12

### 📌 프로젝트 개요

300명 이상이 사용하는 근태 관리 시스템으로 관리자 웹과 직원 웹을 동시에 개발했습니다.
대량의 근태 데이터를 조회하는 테이블에서 성능과 UX를 동시에 만족시키는 구조 설계가 주요 과제였습니다.

### ⚠️ 주요 문제

- Virtualization 도입 시 브라우저 기본 검색(Ctrl+F) 사용이 불가능하여 UX 저하 발생
- 이에 따라 수천 건 이상의 데이터를 한 번에 렌더링해야 했고, 스크롤 및 인터랙션 시 성능 저하 발생

### 💡 해결

- Virtualization 도입을 검토했으나, 데이터 전체 검색 및 접근성이 중요한 서비스 특성을 고려해 UX를 우선 선택
  → 대신 렌더링 구조를 개선하는 방향으로 성능 최적화 진행

- 기존에는 selected를 글로벌 상태로 관리하고, 각 행 하위에 선택 여부를 판단하는 Container 컴포넌트를 두는 구조였음
  → row 수만큼 불필요한 컴포넌트 계층이 증가하여 대량 데이터 렌더링 시 비용이 커지는 문제 발생
  이를 개선하기 위해 Container 컴포넌트를 대신 row 컴포넌트에 메모이제이션을 적용하고, 각 row에는 선택 여부만 나타내는 isSelected boolean props를 전달하도록 변경하여 렌더링 경로를 단순화
  또한 selected를 Set 기반으로 변경하여 각 row의 선택됨 여부를 O(1)에 확인하도록 구조 변경

- hover 및 인터랙션 스타일을 state 기반이 아닌 Tailwind CSS의 group 기능으로 처리하여, 불필요한 상태 관리를 제거하고 렌더링 구조를 경량화

### ✨ 성과

- 불필요한 컴포넌트 계층 제거 및 구조 단순화를 통해 메모리 사용량 감소 및 체감 성능 개선
- 선택 및 인터랙션 처리 시 발생하던 UI 지연을 완화하여 사용자 경험 향상
- Virtualization 없이도 전체 데이터 접근성과 검색 기능을 유지하면서 성능 문제 해결

## 3. 현대중공업 식수 관리 시스템

### 📅 기간

2023.02 ~ 2023.08

### 📌 프로젝트 개요

사업장 식당 단말기에서 발생하는 데이터를 기반으로 식수 데이터를 실시간으로 모니터링하는 관리자 시스템을 개발했습니다.

### ⚠️ 주요 문제

- 다수 사업장에서 발생하는 실시간 데이터를 동시에 처리하며 UI 상태를 즉시 반영해야 하는 요구사항
- 빈번한 데이터 업데이트로 인한 렌더링 비용 증가

### 💡 해결

- MQTT 기반으로 실시간 데이터 스트림을 수신하고, 데이터 단위로 상태 업데이트 처리
- 상태 변경 범위를 최소화하여 전체 리렌더링을 방지

### ✨ 성과

- 수천~수만 명 규모의 식수 데이터를 실시간으로 모니터링할 수 있도록 시각화
- 운영자가 식당 상태를 직관적으로 확인할 수 있는 관리자 대시보드 구축
