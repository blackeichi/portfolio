# 🚀 Key Projects

## 1. 근태관리 + 출입 보안 시스템 웹 리뉴얼

### 📅 기간

2024.01 ~ 2024.03

### 📌 프로젝트 개요

회사 메인 웹 애플리케이션의 레거시 구조 개선과 UI/UX 리뉴얼 프로젝트를 주도적으로 진행했습니다.  
기존 코드베이스는 기능 확장 과정에서 구조적 복잡도가 증가했으며 UI 렌더링 성능 문제와 사용자 경험 이슈가 지속적으로 발생하고 있었습니다.

### ⚠️ 주요 문제

- 클래스 기반 레거시 코드 구조로 인한 유지보수 어려움
- 사용자 인터랙션 시 UI 렉 발생
- UI 디자인 및 인터랙션 일관성 부족

### 💡 해결

#### 아키텍처 개선

- 클래스형 컴포넌트를 함수형 컴포넌트 구조로 전환
- 로딩 / 요청 / 에러 / 재시도 로직을 Custom Hook으로 공통화
- 공통 리소스와 라우트 리소스를 분리하여 프로젝트 구조 재설계

#### 성능 최적화

- 메모이제이션 및 컴포넌트 책임 분리를 통해 불필요한 리렌더링 제거
- dynamic import 기반 코드 스플리팅 적용
- 이미지 최적화 및 렌더링 구조 개선

#### UI/UX 개선

- UI 디자인 시스템 정리 및 스타일 일관성 확보
- 로딩 상태 표시 및 인터랙션 개선

### ✨ 성과

- Lighthouse 성능 점수 **65 → 92** 개선
- 사용자 인터랙션 시 발생하던 UI 렉 문제 해결
- 코드 구조 개선을 통한 유지보수성 향상

## 2. 코나아이 근태 관리 웹 애플리케이션

### 📅 기간

2025.07 ~ 2025.12

### 📌 프로젝트 개요

300명 이상이 사용하는 근태 관리 시스템으로 관리자 웹과 직원 웹을 동시에 개발했습니다.  
대량의 근태 데이터를 조회하는 테이블 렌더링 성능 최적화가 주요 과제였습니다.

### ⚠️ 주요 문제

- 수천 건 이상의 데이터 렌더링으로 인한 성능 저하
- 상태 업데이트 및 렌더링 비용 증가
- Virtualization 도입 시 브라우저 검색 기능(Ctrl+F) 사용 불가

### 💡 해결

- 데이터 처리 로직 개선으로 연산 비용 감소
- useState / useEffect 사용 최소화
- 컴포넌트 책임 분리 및 메모이제이션 적용
- UI 구조 경량화

### ✨ 성과

- 대용량 데이터 환경에서도 안정적인 테이블 렌더링 구현
- 사용자 인터랙션 성능 개선
- 다양한 사용자 상황을 고려한 예외 처리 및 UX 개선

## 3. 현대중공업 식수 관리 시스템

### 📅 기간

2023.02 ~ 2023.08

### 📌 프로젝트 개요

사업장 식당 단말기에서 발생하는 데이터를 기반으로 식수 데이터를 실시간으로 모니터링하는 관리자 시스템을 개발했습니다.

### ⚠️ 주요 문제

- 여러 사업장의 식당 단말기에서 발생하는 데이터를 실시간으로 처리
- 대규모 데이터 기반 UI 상태 업데이트 필요

### 💡 해결

- MQTT 기반 실시간 데이터 수신 로직 구현
- 단말기 데이터 파싱 후 실시간 대시보드 UI 반영
- 식당 운영 상태 및 식수 현황 모니터링 UI 구현

### ✨ 성과

- 수천~수만 명 규모의 식수 데이터를 실시간으로 모니터링할 수 있도록 시각화
- 운영자가 식당 상태를 직관적으로 확인할 수 있는 관리자 대시보드 구축
