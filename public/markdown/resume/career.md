### 📅 기간

2024.01 ~ 2024.03

### 📌 프로젝트 개요

레거시 클래스 컴포넌트 기반 구조로 인해 유지보수 비용이 증가하고,
대량 데이터 렌더링 및 사용자 인터랙션 시 성능 저하 문제가 발생하던 시스템을

### ⚠️ 주요 문제

- 클래스 컴포넌트 중심의 레거시 구조 → 확장성과 유지보수성 저하

- 중복 로직 및 분산된 코드 구조 → 개발 생산성 저하
- UI 디자인 시스템 부재 → 일관성 없는 UX

### 💡 해결

#### 아키텍처 개선

- 로딩 / 요청 / 에러 / 재시도 로직을 Custom Hook으로 공통화
- 공통 리소스와 라우트 리소스를 분리하여 프로젝트 구조 재설계

#### 성능 최적화

- 메모이제이션 및 컴포넌트 책임 분리를 통해 불필요한 리렌더링 제거
- dynamic import 기반 코드 스플리팅 적용
- 이미지 최적화 및 렌더링 구조 개선

#### UI/UX 개선

- 디자인 시스템 정의 및 컴포넌트 공통화
- 로딩/에러 상태 UI 명확화
- 사용자 인터랙션 피드백 개선

### ✨ 성과

- Lighthouse 성능 점수 65 → 92 개선
- 사용자 인터랙션 시 발생하던 UI 렉 제거
- 대량 데이터 처리 시 렌더링 안정성 확보

## 3. 코나아이 근태 관리 웹 애플리케이션

### 📅 기간

2025.07 ~ 2025.12

### 📌 프로젝트 개요

300명 이상이 사용하는 근태 관리 시스템으로 관리자 웹과 직원 웹을 동시에 개발했습니다.
대량의 근태 데이터를 조회하는 테이블에서 성능과 UX를 동시에 만족시키는 구조 설계가 주요 과제였습니다.

### ⚠️ 주요 문제

- Virtualization 도입 시 브라우저 기본 검색(Ctrl+F) 사용이 불가능하여 UX 저하 발생
- 수천 건 데이터를 한 번에 렌더링해야 하며:
  - 스크롤 시 프레임 드랍 발생
  - 선택 및 인터랙션 시 UI 지연 발생
- 기존 구조 문제:
  - selected를 글로벌 상태로 관리하고, 각 행 하위에 선택 여부를 판단하는 Container 컴포넌트를 두는 구조
  - -> row 수만큼 불필요한 컴포넌트 계층이 증가하여 대량 데이터 렌더링 시 비용이 커지는 문제 발생

### 💡 해결

- Virtualization 도입을 검토했으나, 데이터 전체 검색 및 접근성이 중요한 서비스 특성을 고려해 UX를 우선 선택
  → 대신 렌더링 구조를 개선하는 방향으로 성능 최적화 진행

- 렌더링 구조 개선
  Container 컴포넌트 제거 -> row 컴포넌트에 메모이제이션을 적용하고, 각 row에는 선택 여부만 나타내는 isSelected boolean props를 전달하도록 변경하여 렌더링 경로를 단순화
  또한 selected를 Set 기반으로 변경하여 각 row의 선택됨 여부를 O(1)에 확인하도록 구조 변경

- hover 및 인터랙션 스타일을 state 기반이 아닌 Tailwind CSS의 group 기능으로 처리하여, 불필요한 상태 관리를 제거하고 렌더링 구조를 경량화

### ✨ 성과

- 불필요한 컴포넌트 계층 제거 및 구조 단순화를 통해 메모리 사용량 감소 및 성능 개선
- 선택 및 인터랙션 처리 시 발생하던 UI 지연을 완화하여 사용자 경험 향상
- Virtualization 없이도 전체 데이터 접근성과 검색 기능을 유지하면서 성능 문제 해결
