# ⚡ 성능 최적화 전략

## 1. Debouncing

적용 대상:

- 체크박스 연속 클릭
- 리스트 아이템 순서 변경(Drag & Drop)
- 검색 입력 필드
- 자동 저장 기능

효과:

- 불필요한 API 호출 감소
- 네트워크 트래픽 최소화
- 즉각적인 UI 반응과 백그라운드 저장 흐름 분리

## 2. 리스트 가상화

적용 대상:

- 로그 목록 / 테이블 UI

적용 방식:

- `react-window`를 활용해 화면에 보이는 항목만 렌더링
- `react-virtualized-auto-sizer`로 뷰포트 크기에 맞는 높이/너비 자동 계산

효과:

- 대량 데이터 렌더링 시 DOM 노드 수 감소
- 스크롤 성능 개선
- 테이블 UI의 초기 렌더링 부담 완화

## 3. 전역 UI 상태 최적화

- Snackbar, Loading 컴포넌트를 최상단 레이아웃에 배치
- Jotai atom 단위 구독으로 필요한 컴포넌트만 리렌더링

## 4. Dynamic Import

- 모달 등 즉시 필요하지 않은 UI를 지연 로딩
- 초기 번들 크기 감소

## 5. 메모이제이션

- `React.memo`, `useMemo`, `useCallback`을 활용해
  리스트 및 복잡한 컴포넌트 트리의 불필요한 재계산과 리렌더링 방지
